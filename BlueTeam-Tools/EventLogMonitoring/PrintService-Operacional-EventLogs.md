# PrintService Operational Event Log Configuration

## Synopsis
Configures Windows Event Log parameters for the **Microsoft-Windows-PrintService/Operational** channel to support reliable monitoring and forensic analysis of print activity.

## Description
This configuration enables and standardizes logging for the **PrintService Operational channel**, which records detailed printing activity through **Event ID 307** and related events.

The associated `.reg` file configures the following Event Log parameters:

- `AutoBackupLogFiles`
- `Flags`
- Event log file location (`File`)
- Maximum log size (`MaxSize`, `MaxSizeUpper`)
- Retention behavior (`Retention`)

These settings ensure that printing events are preserved with sufficient capacity and retention for:

- **print auditing**
- **incident investigation**
- **digital forensics**
- **compliance and accountability monitoring**

This configuration is intended to support the **EventID307-PrintingAudit.ps1** analysis tool within the EventLogMonitoring toolkit.

# PrintService Operational Logging Overview

The **PrintService Operational log** records granular printer activity generated by the Windows printing subsystem.

Typical events include:

| Event ID | Description |
|--------|-------------|
| 307 | Document successfully printed |
| 805 | Print job processing events |
| 808 | Print job metadata information |

These logs provide critical forensic evidence including:

- username
- document name
- printer used
- number of pages
- print server
- timestamp

# Registry Configuration

The `.reg` configuration applies values under:

```

HKEY_LOCAL_MACHINE
└ SYSTEM
└ ControlSet001
└ Services
└ EventLog
└ Microsoft-Windows-PrintService/Operational

```

Configured values:

| Registry Value | Type | Purpose |
|---------------|------|---------|
| AutoBackupLogFiles | DWORD | Enables automatic log backup when the file reaches capacity |
| Flags | DWORD | Enables operational logging |
| File | REG_SZ | Defines the EVTX storage path |
| MaxSize | DWORD | Maximum log file size |
| MaxSizeUpper | DWORD | Upper limit component of the log size |
| Retention | DWORD | Prevents automatic overwrite |

# Deployment Methods

## Option 1 — Direct Registry Import

1. Save the registry configuration as:

```

PrintService-Operacional-EventLogs.reg

```

2. Execute with administrative privileges:

```

reg import PrintService-Operacional-EventLogs.reg

```

3. Restart the **Print Spooler** service or reboot the system.

---

## Option 2 — Group Policy Deployment (Recommended)

1. Open **Group Policy Management Console**

```

gpmc.msc

```

2. Create or edit a GPO.

3. Navigate to:

```

Computer Configuration
└ Preferences
└ Windows Settings
└ Registry

````

4. Create registry entries using the following values:

| Field | Value |
|------|------|
| Action | Update |
| Hive | HKEY_LOCAL_MACHINE |
| Key Path | SYSTEM\ControlSet001\Services\EventLog\Microsoft-Windows-PrintService/Operational |

Registry entries to create:

| Name | Type | Data |
|------|------|------|
| AutoBackupLogFiles | DWORD | 1 |
| Flags | DWORD | 1 |
| File | REG_SZ | L:\Microsoft-Windows-PrintService-Operational\Microsoft-Windows-PrintService-Operational.evtx |
| MaxSize | DWORD | 09270000 |
| MaxSizeUpper | DWORD | 00000000 |
| Retention | DWORD | FFFFFFFF |

5. Apply policy:

```powershell
gpupdate /force
````

# Security and Forensic Best Practices

* ensure the EVTX storage location is protected with **restricted ACL permissions**
* store logs on **centralized or monitored storage**
* avoid allowing users write access to the log directory
* implement **backup or log forwarding** if used in investigative environments
* maintain archived `.evtx` copies for forensic preservation

---

# Operational Recommendations

For enterprise monitoring environments:

* deploy this configuration through **GPO**
* ensure the storage path exists on all endpoints
* verify the log size is adequate for the organization's print volume
* periodically export or archive `.evtx` logs
